# JSON Map View — 使用说明

## 1. 环境要求

- **Node.js**：建议使用 **22.x**（与仓库内 GitHub Actions 工作流一致），亦可在相近 LTS 版本上验证。
- **包管理**：本仓库使用 **npm**（存在 `package-lock.json`）。

---

## 2. 快速开始

在项目根目录执行：

```bash
npm ci
npm run dev
```

浏览器访问终端提示的本地地址（通常为 `http://localhost:5173`）。开发模式下静态资源 **base** 为 `/`，与生产部署在子路径下的行为可能不同，见下文「部署」。

---

## 3. npm 脚本

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动 Vite 开发服务器（热更新） |
| `npm run build` | `tsc -b` 类型检查 + `vite build` 产出 `dist/` |
| `npm run preview` | 本地预览生产构建结果 |
| `npm run lint` | 运行 ESLint |
| `npm run format` | Prettier 格式化 `src` 等路径 |

## 4. 界面与操作

### 4.1 顶部工具栏

- **加载数据**：主按钮展开菜单，包含三项（各自有独立的文件过滤规则，不符合文件名的条目会被忽略并在工具栏提示）：
  - **Json地图**：选择 `.json` 文件；仅当**文件名**符合「JSON 地图」命名规则时才会加载（见第 5.1 节）。同一时刻全局只能有一个此类地图文件；重复加载会弹出底部提示条，需点 **确认** 关闭，并先在「已加载文件」中移除已有地图。
  - **TUM轨迹**：选择 `.txt` 轨迹文件（见第 5.3 节格式）。
  - **Layer数据**：选择 `.json` 文件；仅当文件名以 **`layer_data.json` 结尾**（不区分大小写）时才会加载（见第 5.2 节）。同一时刻全局只能有一个此类文件；行为与 Json 地图的重复提示类似，点 **确认** 关闭。
- **错误提示**：非法 JSON、名称不符等会在工具栏以文案提示，可点击 **×** 关闭；控制台可能有逐文件详情。

### 4.2 左栏 — 场景树

- 展示合并后的场景层次：**场景**根 → 各 JSON 文档节点 → 其下解析出的图层/对象；若加载了 TUM，另有 **轨迹** 分组。
- **单击行**：选中对应节点（与 3D 视口中拾取共用同一 `nodeId`）；有子节点时单击也会切换展开/折叠。
- **眼睛图标**：切换该节点子树在 **3D 视口** 中的显示/隐藏；场景树中仍列出，仅影响渲染。
- 开启**区域筛选**时，与当前筛选不一致的区域在树中可能以较弱样式显示（与视口过滤一致）。

### 4.3 左栏 — 已加载文件

- 按**加载时间**混合列出所有已加载的 JSON 文档与 TUM 轨迹。
- **单击某 JSON 行**：将其设为**活动文档**（影响右侧属性与区域列表的数据源）。
- **移除**：从会话中卸载该文件；移除当前活动文档时会自动切换到剩余文档之一（若有）。

### 4.4 中栏 — 3D 视口

- **轨道控制**（[Three.js OrbitControls](https://threejs.org/docs/#examples/en/controls/OrbitControls)，与当前工程配置一致）：
  - **左键拖动**：旋转视角。
  - **中键拖动**：旋转视角（与左键相同）。
  - **滚轮**：缩放（靠近/远离场景）。
  - **右键拖动**：平移。
- **单击对象**：选中对应场景节点（与场景树共用 `nodeId`）。
- **单击空白**：清除选中。**距离测量模式**开启时，单击空白**不会**清除选中（避免干扰测距）；测距点仍可通过射线落在场景几何或 **y=0** 地面平面上放置（见下文测距说明）。
- **网格与辅助**：地面**背景网格**与**地图坐标系**（原点轴与刻度）便于对齐；二者在**测距射线拾取**中会被忽略（不会把点卡在网格/轴线上），拾取逻辑见 `src/utils/measurePick.ts`。
- **右下角工具栏**：圆形按钮可展开 **3D 视口工具**。
  - **距离测量**：开关测距模式。开启后视口下方会显示简短操作提示。
    - **左键**：依次放置第一个点、第二个点；线段与距离（米，两位小数）显示在两点中点；第三点起重新开始一段测距。
    - **测距时**：左键由测距逻辑优先处理（用于放点）；旋转请用**中键**，缩放用**滚轮**，平移用**右键**。
    - **Esc** 或再次点击工具栏中的测距按钮：退出测距并清除测距点。
    - 测距距离为**场景坐标系**下两点欧氏距离（与 HD Map / Layer 使用的 Y-up 场景空间一致）。
    - 端点标记的**屏幕视觉大小**会随相机距离大致保持稳定（与坐标轴刻度标签同类缩放策略）。
- **`road_links` 渲染规则**（道路边界线节点）：
  - 场景树中，`roadLink` 下的「道路边界线」为叶子节点（不再展开 `ref_traj_points` / 左右边界子节点）。
  - `ref_traj_points`：渲染和点选逻辑保持原有行为；受「点渲染模式」开关影响（线/点）。
  - `road_boundary_left_points` 与 `road_boundary_right_points`：始终只渲染为小球，不连线，且不支持点选。

### 4.5 右栏 — 属性与区域

- **属性**：展示当前选中节点相关的结构化字段（非原始 JSON 全文）；内容依赖节点类型与 `payload`。
- **区域**：当**活动文档**为包含 `regionList` 的地图 JSON 时显示；列出各区域的名称、id、高度与区域连接等字段。
- **筛选**：某一区域行的 **筛选** 按钮用于在视口中只显示带相同 `regionID` 的几何；**无** `regionID` 的对象始终显示。再次点击已高亮的「筛选」可关闭筛选。
- **`road_links` 点渲染模式开关**：仅作用于 `ref_traj_points`，不影响 `road_boundary_left_points` / `road_boundary_right_points`（后二者始终为点球显示）。

## 5. 支持的文件格式

### 5.1 Json地图（经「Json地图」菜单加载的 `.json`）

- 须为扩展名 **`.json`**，且**基名**（去掉 `.json` 后的部分，不区分大小写）满足：其中出现 **`json_map` 这一段**，且该段**前**为行首或非字母数字、**后**不能紧跟小写字母（用于排除如 `json_mapper` 等误匹配）。示例：`foo_json_map.json`、`2_json_map (1).json`、`json_map_backup.json`。
- **解析顺序**（对合法 JSON）：先判断是否 HD Map 根结构，是则走地图图层（道路、车位、箭头等）；否则判断是否 Layer 导出根结构（见 5.2），是则走 Layer 解析；再否则按**有界深度与广度**生成通用「结构树」占位视图。
- **HD Map**：根对象包含 `mapJsonToScene` 所识别的字段时按地图解析。部分键（如 `header`、`mapId` 等）被有意跳过，不生成节点。
- **唯一性**：任意时刻已加载列表中只能有 **一个** 文件名符合上述「Json地图」命名规则的文件；换图请先在「已加载文件」中移除当前地图。

### 5.2 Layer 数据（经「Layer数据」菜单加载的 `.json`）

- 文件名须以 **`layer_data.json` 结尾**（不区分大小写），例如 `0407_093132_layer_data.json`。
- 根对象若包含以下任一数组字段，会按 Layer 导出解析：`all_boundary_pts`、`all_lane_line_pts`、`center_nodes`（点坐标为 `{x,y,z}`，与 HD Map 相同的车体系 → 场景坐标映射）。边界/车道线点云与中心节点图等在 3D 视口中绘制；详情见 [架构设计](./ARCHITECTURE.md) 中 `layerDataToScene`。
- 若名称符合但内容不匹配上述结构，会退化为**通用结构树**（与 5.1 相同策略）。
- **唯一性**：任意时刻只能有 **一个** `*layer_data.json` 文件处于已加载状态。
- **与 Json 地图的布局**：`json_map` 类地图与 `layer_data.json` 在场景合并时占用**同一网格槽位**，便于叠在同一坐标系下查看（见 `buildSceneTree`）。

### 5.3 TUM 轨迹（`.txt`）

每行 **8 个**用空白分隔的数值：

`timestamp tx ty tz qx qy qz qw`

- 时间戳与位姿（位置 + 四元数）；以 `#` 开头的行视为注释；空行忽略。
- 至少 **2** 条有效姿态才会视为解析成功。
- 轨迹点位置会经与地图相同的 **车体系 → 场景** 坐标映射后，连成折线显示在 **轨迹** 下；多条轨迹使用不同颜色区分。
- **同名文件**不可重复加载（以文件名为准）。

## 6. 常见问题（FAQ）

**Q：页面空白或静态资源 404（部署在线上时）**  
A：GitHub Pages 项目站使用 `/仓库名/` 前缀。请使用 CI 中设置了 `GITHUB_REPOSITORY` 的构建产物部署；不要在未设置该变量的本地 `npm run build` 结果上直接当作子路径站点使用。详见 [github-pages-deploy.md](./github-pages-deploy.md)。

**Q：为什么第二次选择 Json 地图或 Layer 文件没有加载？**  
A：Json 地图与 Layer 数据各自全局只允许一个符合命名规则的文件。请先移除「已加载文件」中已有项，并留意底部浮动提示条（点 **确认** 关闭）。

**Q：选了 `.json` 却提示文件名不符合？**  
A：Json 地图入口只接受含 **`json_map`** 段且符合规则的基名；Layer 入口只接受以 **`layer_data.json`** 结尾的文件。请从对应菜单选择，或按要求改名。

**Q：JSON 很大时浏览器卡顿？**  
A：通用 JSON 模式有深度与广度上限；地图模式请按需只加载必要文件。极端大的文件仍可能导致浏览器压力，可尝试关闭其他标签页或分批检查。

**Q：选中场景树节点后相机是否会移动？**  
A：在场景树中选中节点时，会请求将轨道控制器目标对准该节点在 3D 中的范围（具体以 `CameraFocusSync` 实现为准）。

**Q：测距模式下左键拖不动视角？**  
A：测距开启时左键用于放置测距点。旋转请用**中键拖动**，缩放用**滚轮**，平移用**右键拖动**。

**Q：`npm run build` 提示 chunk 大于 500 kB？**  
A：这是 Vite/Rollup 的体积提示，不是错误；构建仍成功。若需优化首包，可考虑懒加载或拆包；否则可忽略。
