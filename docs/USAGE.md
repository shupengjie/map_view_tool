# JSON Map View — 使用说明

## 1. 环境要求

- **Node.js**：建议使用 **22.x**（与仓库内 GitHub Actions 工作流一致），亦可在相近 LTS 版本上验证。
- **包管理**：本仓库使用 **npm**（存在 `package-lock.json`）。

---

## 2. 快速开始

在项目根目录执行：

```bash
npm ci
npm run dev
```

浏览器访问终端提示的本地地址（通常为 `http://localhost:5173`）。开发模式下静态资源 **base** 为 `/`，与生产部署在子路径下的行为可能不同，见下文「部署」。

---

## 3. npm 脚本

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动 Vite 开发服务器（热更新） |
| `npm run build` | `tsc -b` 类型检查 + `vite build` 产出 `dist/` |
| `npm run preview` | 本地预览生产构建结果 |
| `npm run lint` | 运行 ESLint |
| `npm run format` | Prettier 格式化 `src` 等路径 |

## 4. 界面与操作

### 4.1 顶部工具栏

- **加载本地 JSON…**：打开系统文件选择器，可选多个 `.json` 文件。非合法 JSON 会在控制台输出详情，工具栏可能显示汇总提示。
- **加载 TUM 轨迹**：选择 `.txt` 轨迹文件（见第 5.2 节格式）。
- **错误提示**：加载问题会在工具栏以文案提示，可点击 **×** 关闭。

### 4.2 左栏 — 场景树

- 展示合并后的场景层次：**场景**根 → 各 JSON 文档节点 → 其下解析出的图层/对象；若加载了 TUM，另有 **轨迹** 分组。
- **单击行**：选中对应节点（与 3D 视口中拾取共用同一 `nodeId`）；有子节点时单击也会切换展开/折叠。
- **眼睛图标**：切换该节点子树在 **3D 视口** 中的显示/隐藏；场景树中仍列出，仅影响渲染。
- 开启**区域筛选**时，与当前筛选不一致的区域在树中可能以较弱样式显示（与视口过滤一致）。

### 4.3 左栏 — 已加载文件

- 按**加载时间**混合列出所有已加载的 JSON 文档与 TUM 轨迹。
- **单击某 JSON 行**：将其设为**活动文档**（影响右侧属性与区域列表的数据源）。
- **移除**：从会话中卸载该文件；移除当前活动文档时会自动切换到剩余文档之一（若有）。

### 4.4 中栏 — 3D 视口

- **轨道控制**：拖拽旋转、滚轮缩放等（行为由 OrbitControls 提供）。
- **单击对象**：选中对应场景节点。
- **单击空白**：清除选中。
- **网格与辅助**：地面网格与坐标系辅助显示，便于对齐。

### 4.5 右栏 — 属性与区域

- **属性**：展示当前选中节点相关的结构化字段（非原始 JSON 全文）；内容依赖节点类型与 `payload`。
- **区域**：当**活动文档**为包含 `regionList` 的地图 JSON 时显示；列出各区域的名称、id、高度与区域连接等字段。
- **筛选**：某一区域行的 **筛选** 按钮用于在视口中只显示带相同 `regionID` 的几何；**无** `regionID` 的对象始终显示。再次点击已高亮的「筛选」可关闭筛选。

## 5. 支持的文件格式

### 5.1 JSON

- 扩展名为 `.json` 或 MIME 为 `application/json` 的文件会被尝试解析。
- **通用 JSON**：若根对象不符合 HD Map 特征，将按深度与每层子节点数量上限生成占位「结构树」（便于浏览大型或任意 JSON）。
- **HD Map / 地图 JSON**：根对象包含 `mapJsonToScene` 所识别的字段时，按地图图层解析为道路、车位、箭头等几何。部分键（如 `header`、`mapId` 等）被有意跳过，不生成节点。
- **地图包命名**：文件名以 **`json_map.json` 结尾**（例如 `foo_json_map.json`）的文件全局仅能加载 **一个**；需更换地图时请先在「已加载文件」中移除当前地图再加载新文件。

### 5.2 TUM 轨迹（`.txt`）

每行 **8 个**用空白分隔的数值：

`timestamp tx ty tz qx qy qz qw`

- 时间戳与位姿（位置 + 四元数）；以 `#` 开头的行视为注释；空行忽略。
- 至少 **2** 条有效姿态才会视为解析成功。
- 轨迹点位置会经与地图相同的 **车体系 → 场景** 坐标映射后，连成折线显示在 **轨迹** 下；多条轨迹使用不同颜色区分。
- **同名文件**不可重复加载（以文件名为准）。

## 6. 常见问题（FAQ）

**Q：页面空白或静态资源 404（部署在线上时）**  
A：GitHub Pages 项目站使用 `/仓库名/` 前缀。请使用 CI 中设置了 `GITHUB_REPOSITORY` 的构建产物部署；不要在未设置该变量的本地 `npm run build` 结果上直接当作子路径站点使用。详见 [github-pages-deploy.md](./github-pages-deploy.md)。

**Q：为什么第二次选择 `xxx_json_map.json` 没有加载？**  
A：同一时间只允许一个以 `json_map.json` 结尾的地图文件；请先移除已加载的那一项，并留意顶部浮动提示说明。

**Q：JSON 很大时浏览器卡顿？**  
A：通用 JSON 模式有深度与广度上限；地图模式请按需只加载必要文件。极端大的文件仍可能导致浏览器压力，可尝试关闭其他标签页或分批检查。

**Q：选中场景树节点后相机是否会移动？**  
A：在场景树中选中节点时，会请求将轨道控制器目标对准该节点在 3D 中的范围（具体以 `CameraFocusSync` 实现为准）。
